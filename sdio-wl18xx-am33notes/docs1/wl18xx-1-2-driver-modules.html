How to initialize/load the wl18xx driver when it 's build as module?

Intellectual 740 points
yuqing cai

Recently, I am trying to integrate the "WL1835MOD" driver to my OMAP3 system(running linux-3.2.1),  
linux-3.2.1 source tree doesn't contain the wl18xx driver source, according to the WL18xx System Build Scripts,
I generated the following components as modules from the compat-wireless source and install them in my
root file system:

compat.ko, wl18xx.ko, wlcore.ko, wlcore_sdio.ko, mac80211.ko, cfg80211.ko
wilink8 firmware
crda
hostap
wpa_supplicant
iw
libnl
openssl
uim

here is my questions:
1) How to pass the initialization data "platfrom_data structure" to the wl18xx driver(driver is build as a module)?
According to WL18xx Platform Integration Guide, it use the function "wl12xx_set_platform_data" in the board 
initialization file(such as "board-omap3beagle.c" or "board-omap3evm.c")to initialize the driver, in this way,  it seems
that the driver has to build in kernel but not a module then initialize it when the system boot.
So what is the correct way to initialize the driver when it is a module?

2) How to load the driver modules after the system is finish booting. The modules is a driver stack, so it must
load as an sequence, is there a boot scripts in the building system?

    Hi,

    1. wl12xx_set_platform_data() is defined in <kernel>/drivers/net/wireless/wl12xx/wl12xx_platform_data.c - the compilation depends on "CONFIG_WL12XX_PLATFORM_DATA" (not part of the wl12xx modules).

    2. Yes, you are correct. Please use the below order of loading:

    1. compat.ko
    2. cfg80211.ko
    3. mac80211.ko
    4. wlcore.ko
    5. wl18xx.ko
    6. wlcore_sdio.ko

    Regards,
    Gigi Joseph.
